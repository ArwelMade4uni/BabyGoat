Insecure Direct Object Referencing


Insecure Direct Object Referencing (IDOR) is a common type of vulnerability that isn't exclusive to a certain language but rather relies on the security behind the website. IDOR is when a user can find different webpages, images or even log into a different account by using the internal file name and changing it via sending altered packages or by simply altering the URL. An example is when a potential attacker wants to log into another persons account, they would be able to check the internal names of variables or even the URL to find a weakspot. If the attacker sees something they can exploit like ID=100 they can change their ID to a different number and try to send it off as a legitimate send request to the server. The server, if it sees the request as a legitimate one, will allow access to the account that the ID was changed to. This exploit mainly targets websites with many accounts like banks, Facebook, Twitter or any other website using log ins.

Testing IDOR

To try out this exploit try typing in the URL 'IDOR2.html' inplace of the original 'IDOR1.html'. You will be taken to the hidden webpage.
